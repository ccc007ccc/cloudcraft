# CloudCraft 重构实施规格

本文是基于实际进游戏反馈的新规格。旧 `SPEC_CLOUDCRAFT_IMPLEMENTATION.md` 只记录上一轮实现闭环，不再代表最终目标。

勾选规则：

- 只有代码、资源、文档和测试证据都足够时才能勾选。
- 游戏体验项必须有客户端或 GameTest / 手工烟测记录。
- 贴图项必须有 APX 源、预览、正式 PNG 和视觉检查记录。
- 任何临时方案必须标注“临时”，不能伪装成最终设计。

## 0. 当前问题基线

- [ ] `cloud-preview` 暗部被改成硬色块，缺少立体感和层次。
- [ ] 盔甲资产不在游戏内可见，设计状态不清。
- [ ] 喷气背包穿在背槽后第三人称不可见。
- [ ] 启动或进入世界时出现实验性设置警告。
- [ ] 背槽喷气背包死亡后不会按玩家掉落逻辑掉落。
- [ ] 喷气背包物品图标没有类似耐久条的气压储量显示。
- [ ] 喷气背包鼠标悬浮没有显示气压和飞行参数。
- [ ] 气态转换器没有 GUI，不像科技线机器。
- [ ] 气态转换器承担维度入口职责，职责错误。
- [ ] 云维度入口方式需要重新设计。
- [ ] 云维度返回主世界方式需要重新设计。
- [ ] 云维度坐标和高度设定不符合“高天云域”。
- [ ] 云维度生成不应是连绵大地，应更接近末地空岛。
- [ ] 不同云属性质应影响空岛连续性、大小、厚度和危险度。
- [ ] 喷气背包推进时需要粒子效果。
